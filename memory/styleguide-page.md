The `lifesystem` client (`client/src`) has a `/styleguide` route ([pages/StyleGuide.tsx](client/src/pages/StyleGuide.tsx)) that showcases every shared component with a live preview plus a copy-paste code snippet via the `CodeBlock` component. Components so far: Container, Button, Badge, Input, Card, Alert, Tabs, Switch, Checkbox, Avatar, Spinner, Progress, DatePicker, Textarea, Select, RadioGroup, Slider, Chip, EmptyState, Rating, Breadcrumbs, Pagination, Accordion, Tooltip, DropdownMenu, Modal, Drawer. Modal & Drawer render via `createPortal` and share `components/useOverlay.ts` (`useOverlayBehavior` — Escape-to-close + body-scroll-lock); Drawer stays mounted and slides via `translate-x`. DropdownMenu & Accordion self-manage open state; Select reuses DatePicker's click-outside + trigger styling. `Container` (`components/Container.tsx`) is the shared layout wrapper — centered, `max-w-6xl`, `px-4 sm:px-6`, with an `as` prop; the Navbar and all page content route through it so edges align (don't reintroduce ad-hoc `max-w-*/mx-auto` wrappers). Tabs/Switch/Checkbox/DatePicker are uncontrolled-capable (internal state with optional controlled `value`/`checked` + `onChange`). DatePicker (`components/DatePicker.tsx`) supports `mode="single" | "range"`, ported from munro-locator's date-picker; single emits `"YYYY-MM-DD"`, range emits `{ start, end }`. Also supports `minDate`/`maxDate` (ISO bounds), `disabledDates` (greyed, struck through), and `errorDates` (red, blocked) — both accept an ISO string array or a `(date: Date) => boolean` predicate (`DateMatcher` type). Disabled/error days are non-selectable. The calendar header cycles day → month → year views (click the title; arrows page by month/year/decade accordingly), and a "Today" footer button jumps the view back to the current month. The homepage (`pages/Home.tsx`) is intentionally minimal — just "Homepage" + a link to the style guide.

When adding or restyling a shared component, update the style guide to match. Styling follows [[design-language]].
